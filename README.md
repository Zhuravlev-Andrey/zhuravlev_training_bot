# migrations/

Здесь хранятся SQL-миграции — версионированные изменения схемы БД.

## Зачем

При изменении схемы БД (новая таблица, новая колонка, изменение типа)
без миграций приходится вручную выполнять SQL на продакшн-базе.
Это опасно: легко забыть шаг, потерять историю, сломать данные.

Миграции решают это: каждое изменение — отдельный файл с номером версии.
Инструмент применяет только те, которые ещё не были применены.

## Инструмент

Рекомендуется [yoyo-migrations](https://ollycope.com/software/yoyo/latest/):
```
pip install yoyo-migrations
```

## Соглашение об именовании файлов

```
NNNN_короткое_описание.sql
```

Примеры:
```
0001_initial_schema.sql
0002_add_schedule_entry.sql
0003_add_workout_set_comment.sql
```

## Применение миграций

```bash
# Применить все новые миграции
yoyo apply --database $DATABASE_URL ./migrations

# Откатить последнюю
yoyo rollback --database $DATABASE_URL ./migrations

# Посмотреть статус
yoyo list --database $DATABASE_URL ./migrations
```

## Важно

- Никогда не редактируйте уже применённую миграцию.
- Новые изменения — всегда новый файл с следующим номером.
- Деструктивные операции (DROP TABLE, DROP COLUMN) — отдельный файл
  с комментарием о том, что данные будут потеряны.
